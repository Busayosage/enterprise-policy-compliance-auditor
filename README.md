# Enterprise Policy Compliance Auditor

A Streamlit-based RAG (Retrieval-Augmented Generation) application that allows users to upload corporate policy PDFs and ask compliance questions in natural language. Every answer is grounded strictly in the uploaded document, with source snippets displayed for verification.

## Business Problem

Compliance teams, HR departments, and operations teams regularly need to search through lengthy policy documents to answer specific questions. Manually reviewing these documents is slow, inconsistent, and creates risk when clauses are missed or misinterpreted. Organizations need a faster, more reliable way to query their own policies.

## Solution

This application uses retrieval-augmented generation to solve the problem. When a user uploads a policy PDF, the app extracts the text, splits it into chunks, and stores the chunks as vector embeddings in ChromaDB. When the user asks a question, the app retrieves the most relevant sections from the vector store and sends them to NVIDIA NIM (LLaMA 3.1 70B Instruct) with a strict compliance auditor prompt. The model generates an answer using only the retrieved context. If the document does not contain enough evidence, the model explicitly refuses to answer rather than guessing.

## Key Features

- **PDF Upload and Indexing** — Upload any policy PDF and index it into a vector database with one click.
- **Document Chunking** — Text is split into overlapping chunks for accurate retrieval.
- **ChromaDB Vector Storage** — Document embeddings are stored locally in a persistent ChromaDB instance.
- **NVIDIA NIM Answer Generation** — Answers are generated by LLaMA 3.1 70B Instruct via NVIDIA NIM.
- **Source Snippets** — Every answer includes the exact document passages used, with page numbers.
- **Strict Document Grounding** — The AI is constrained to answer only from the uploaded document content.
- **Hallucination Refusal** — When the document does not contain enough evidence, the model responds: "I cannot find enough evidence in the uploaded policy to answer that."

## Architecture

```
User uploads PDF
  → PDF text extraction (PyPDFLoader)
  → Text chunking (RecursiveCharacterTextSplitter)
  → Embedding generation (Sentence Transformers)
  → ChromaDB vector storage

User asks question
  → Relevant chunks retrieved from ChromaDB
  → NVIDIA NIM generates grounded answer
  → Answer + source snippets displayed in Streamlit
```

## Example Questions

- "What does the policy say about conflicts of interest?"
- "What should an employee do if they are offered a gift?"
- "What are the consequences of breaching the code?"
- "What does the policy say about remote working from another country?"

The last question serves as a hallucination-control test. If the uploaded policy does not mention remote working from another country, the app should refuse to answer rather than fabricating a response. This demonstrates the strict grounding behavior of the system.

## Local Setup

Run these commands in Windows PowerShell:

```powershell
python -m venv venv
venv\Scripts\activate
python -m pip install -r requirements.txt
copy .env.example .env
python -m streamlit run app.py
```

After running `copy .env.example .env`, open the `.env` file and add your NVIDIA API key before starting the app.

## Environment Variables

This project requires one environment variable. Add it to your `.env` file:

```
NVIDIA_API_KEY=your_nvidia_api_key_here
```

You can obtain a free API key from [build.nvidia.com](https://build.nvidia.com).

**Important:** The `.env` file contains your real API key and should never be committed to GitHub. It is already included in `.gitignore`. Only `.env.example` (which contains a placeholder) is safe to commit.

## Current Limitations

- Only PDF input is currently supported.
- Response speed depends on NVIDIA API and model availability.
- Source page numbering may depend on the structure and metadata of the uploaded PDF.
- This is a portfolio and MVP project. It is not a replacement for professional legal or compliance advice.
